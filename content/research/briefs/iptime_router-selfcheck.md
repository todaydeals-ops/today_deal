# iptime 공유기 자가점검 조사

**주제:** iptime 공유기 인터넷 끊김·느림·접속 불가 시 점검 순서  
**조사 기간:** 2026-07-21  
**대상:** 국내 무선 공유기 사용자 (현행 iptime 모델)

---

## 1. 재부팅 순서와 대기 시간

### 문제 발생 시 첫 번째 조치: 재부팅

인터넷이 끊기거나 느릴 때, 가장 일반적인 해결책은 **모뎀과 공유기의 재부팅**이다. 이 방법만으로도 80~90% 정도의 문제가 해결되는 것으로 보고된다.

**절차:**
1. 모뎀과 공유기의 전원 플러그를 **동시에** 분리
2. **1~2분 대기** (공유기와 모뎀의 캐시 및 메모리 초기화)
3. 모뎀부터 먼저 전원을 연결하고, 안정화 후(약 1분) 공유기 전원 연결

모뎀을 먼저 연결하는 이유는 모뎀이 WAN 신호를 안정화시킨 후 공유기가 인식하도록 하기 위함이다.

---

## 2. 관리자 페이지 접속 및 비밀번호 관리

### 관리자 페이지 진입 방법

**기본 접속 주소:** `192.168.0.1`

웹 브라우저의 주소창에 위 주소를 입력하면 로그인 창이 나타난다. 다만 다음의 경우 주소가 다를 수 있다:
- 통신사 제공 공유기(KT/SKB/LG 등) 아래에 iptime을 이중 연결한 경우
- 메시 네트워크나 AP 모드로 설정된 경우

이런 경우 **Windows 명령 프롬프트에서 `ipconfig` 실행** → "Default Gateway" 항목을 확인 → 그 IP 주소로 접속해야 한다.

### 기본 비밀번호

- **기본값:** admin / admin
- **주의사항:** 초기 설치 시 이미 사용자가 관리자 계정을 설정했다면, 그 때 설정한 비밀번호를 사용해야 한다.

### 비밀번호 분실 시 초기화

비밀번호를 잊어버렸으면 **공장 초기화(RST 버튼)**를 통해서만 진행할 수 있다. 이 과정에서 모든 설정값이 초기화되므로, 초기화 후 네트워크명(SSID)과 비밀번호를 다시 설정해야 한다.

---

## 3. 공장 초기화 (RST 버튼)

### 버튼 위치 및 누르는 방법

**위치:** 공유기 뒷면이나 옆면 (작은 오목한 구멍 형태)  
**표시:** "RST" 또는 "RST/WPS"라고 인쇄됨

**누르는 시간:**
- **1~2초 (단순 재부팅):** 설정값 유지, 기존 비밀번호·SSID 그대로
- **약 10초 (공장 초기화):** CPU LED가 **빠르게 깜빡일 때까지** 계속 누르기 → 모든 설정 삭제

### 도구 사용

손톱으로 누르기 어려우므로 **볼펜, 이쑤시개, 클립** 등 뾰족한 도구를 사용한다.

### 초기화 후 절차

1. CPU LED가 빠르게 깜빡이는 것을 확인 후 손 떼기
2. 공유기가 자동으로 재부팅될 때까지 기다리기 (1~2분)
3. 다시 접속해 관리자 페이지 설정 (192.168.0.1, admin/admin)

---

## 4. 무선 채널 및 대역폭 설정

### 채널 변경의 필요성

같은 2.4GHz나 5GHz 대역을 사용하는 이웃 공유기들과 **주파수 간섭**이 발생할 수 있다. 이는 속도 저하나 연결 불안정으로 나타난다.

### 채널 자동 선택 방법

1. 관리자 페이지(192.168.0.1) 접속
2. 메뉴 → **기본설정** → **무선설정/보안**
3. "2.4GHz 설정" 또는 "5GHz 설정"에서 **[채널 검색]** 버튼 클릭
4. 시스템이 자동으로 간섭이 적은 빈 채널을 찾아 적용

### 대역폭 설정

무선 설정 메뉴에서 대역폭을 다음 중 선택할 수 있다:
- **자동 (권장):** 시스템이 자동으로 최적 값 설정
- **20MHz:** 안정성 우선 (느린 속도)
- **40MHz:** 속도 우선 (간섭에 민감)

### 2.4GHz vs 5GHz 선택 가이드

| 대역 | 특징 | 추천 용도 |
|---|---|---|
| **2.4GHz** | 신호 범위 넓음, 벽 투과 잘됨 | 거리 멀고 장애물 많은 곳 |
| **5GHz** | 간섭 적음, 빠른 속도 | 가까운 거리, 고속 필요시 |

대부분의 최신 iptime 공유기는 **듀얼밴드(2.4GHz + 5GHz)** 동시 지원하므로, 상황에 맞춰 기기를 할당하면 된다.

## 5. LED 표시등 의미

iptime 공유기의 LED는 현재 상태를 나타내는 신호다.

### 주요 LED 종류

- **전원(Power) LED:** 공유기 전원 상태
- **인터넷(Internet) LED:** 모뎀과의 WAN 연결 상태
- **무선(Wireless/WiFi) LED:** 무선 신호 상태
- **CPU LED:** 공유기 작동 상태

### LED 상태 읽기

- **켜짐(정상):** 해당 기능 정상 작동
- **깜빡임(통신 중):** 데이터 송수신 중
- **꺼짐:** 해당 기능 비활성 또는 오류

### WPS 버튼 사용 시

WPS 버튼을 누르면:
1. 공유기의 LED가 **2분간 점등** 상태 유지
2. 그 동안 다른 기기(스마트폰, 노트북 등)의 WPS 기능으로 신청
3. 자동으로 비밀번호 입력 없이 연결됨

## 6. 펌웨어 업그레이드

### 업그레이드 방법 및 주의사항

**진행:**
1. 관리자 페이지(192.168.0.1) 접속
2. 시스템 설정 또는 관리 메뉴에서 펌웨어 확인
3. 새 버전이 있으면 업데이트 버튼 클릭
4. **완료될 때까지 전원을 끄면 안 됨**

**주의:**
- 업그레이드 중 공유기가 자동 재부팅되면서 **인터넷이 1~5분 끊김**
- 업그레이드 완료 후에도 재부팅으로 기존 설정이 유지되는지 확인

---

## 7. 특정 기기만 연결되지 않을 때

### 진단 순서

1. **다른 기기 테스트:** 같은 네트워크에 다른 기기(스마트폰 등)가 연결되는지 확인
   - 다른 기기도 안 되면 → 공유기 문제
   - 다른 기기는 되면 → 특정 기기 문제

2. **기기의 무선 설정 초기화**
   - 스마트폰/노트북의 Wi-Fi 설정에서 해당 네트워크 "제거" 후 다시 연결

3. **보안 암호화 방식 확인**
   - 공유기 설정에서 WPA2 또는 혼합 모드 사용 (WPA3는 구형 기기 미지원)
   - 192.168.0.1 → 무선설정/보안 → 인증방법 확인

4. **2.4GHz/5GHz 대역 확인**
   - 구형 기기는 5GHz 미지원 가능 → 2.4GHz 대역만 사용
   - 이중 SSID인 경우 각 대역에 맞춰 연결

### 해결 안 될 때

- 공유기를 단순 재부팅(전원 분리 1분 후 재연결)
- 공유기 RST로 공장 초기화 후 다시 설정

---

## 조사 한계 및 추가 확인 필요 항목

### 확인된 정보

- 관리자 페이지 접속: 192.168.0.1 (기본값)
- RST 버튼: 약 10초 (CPU LED 깜빡일 때까지)
- 재부팅 대기 시간: 1~2분
- 무선 채널 설정: 자동 채널 검색 가능
- 2.4GHz/5GHz 특성: 범위 vs 속도 트레이드오프

### 미확인 항목 (공식 매뉴얼 접근 제약)

- **펌웨어 업그레이드 세부 절차:** 모델별로 다를 수 있음 → 사용자 후기 기반 수집
- **특정 모델의 LED 표시등 색상:** iptime 모델 다양성으로 일괄 적용 어려움
- **신호 약할 때 공유기 배치:** 일반 이론(높이, 중앙 배치 등)만 확인, iptime 특화 가이드 없음
