# 외장하드·USB 메모리 인식 안 될 때 자가점검

## 조사 개요

외장하드와 USB 메모리가 컴퓨터에 인식되지 않을 때 독자가 직접 점검할 수 있는 절차를 Windows, macOS 기준으로 정리. 데이터 손실이 우려되는 주제이므로, 안전한 조치(포트 변경·케이블 재연결)와 위험한 조치(포맷·재부팅 없이 분리)를 명확히 구분.

---

## 1. 기본 점검 — 하드웨어 연결

### 케이블과 포트 확인
- **육안 점검**: 케이블이 양쪽 끝 모두 완전히 꽂혀 있는지 확인. USB 포트 금속 부분에 눈에 띄는 손상(휘어짐, 녹슨 흔적)이 없는지 보기.
- **다른 포트 시도**: 같은 컴퓨터의 다른 USB 포트에 연결하여 재시도. 특히 USB 허브가 아닌 컴퓨터 본체 포트 우선.
- **전원 공급 확인**: 외장하드는 전원이 공급되지 않는 USB 허브를 통한 연결보다 컴퓨터에 직접 연결할 것. 노트북은 측면 포트보다 후면 포트, 또는 독립형 전원 어댑터 가능 제품 우선.
- **다른 PC 테스트**: 가능하면 다른 컴퓨터에도 연결해볼 것. 다른 PC에서도 인식 안 되면 드라이브 자체의 문제일 가능성 높음.

**출처**: Microsoft Learn, USB 장치 인식 실패 오류 해결

---

## 2. Windows — 장치관리자와 디스크 관리 점검

### 단계 1: 장치관리자에서 하드웨어 인식 확인
1. Windows 시작 > 검색창에 "장치 관리자" 입력 > 실행
2. 목록에서 "디스크 드라이브" 항목 찾기
3. 외장하드 이름이 있는가?
   - **있으면**: 하드웨어는 인식됨. 다음 단계로.
   - **없으면**: 컴퓨터가 기기 자체를 감지 못함. 케이블/포트 재확인 필수.

### 단계 2: USB 컨트롤러 상태 확인
장치관리자에서 "범용 직렬 버스 컨트롤러" 또는 "USB 호스트 컨트롤러" 항목을 펼쳐서, 항목 옆에 노란 느낌표(!)나 아래향 화살표가 없는지 확인.
- **느낌표 있으면**: 드라이버 오류. 우클릭 > "드라이버 업데이트" 시도.
- **아래향 화살표(사용 안 함)이면**: 우클릭 > "사용"을 선택하여 활성화.

**출처**: Microsoft Learn

### 단계 3: 디스크 관리에서 드라이브 확인
1. Windows 시작 > 검색에 "디스크 관리" 입력
2. 목록에서 외장하드 파티션이 보이는가?
   - **보이는데 드라이브 문자가 없으면(예: "F:" 표시 없음)**: 다음 절차로 진행.
   - **안 보이면**: 디스크 관리 진행 중 화면 새로고침 필요. 상단 메뉴 > "작업" > "디스크 다시 검사" 클릭 후 대기.

### 단계 4: 드라이브 문자 할당
디스크 관리에서 외장하드 파티션이 보이지만 내 PC에 드라이브가 표시되지 않으면:
1. 파티션을 우클릭
2. "드라이브 문자 및 경로 변경" 선택
3. "추가" 버튼 클릭
4. 사용 가능한 문자(보통 D, E, F 등) 선택
5. "확인" 클릭

이 후 내 PC에 드라이브가 나타나야 함.

**출처**: Microsoft Q&A

### 단계 5: 드라이버 재설치 (고급)
위의 모든 절차 후에도 인식 안 되면:
1. 장치관리자에서 "범용 직렬 버스 컨트롤러" 아래 모든 항목을 우클릭 > "제거"
2. 컴퓨터 재시작
3. USB 컨트롤러가 자동으로 재설치됨

## 3. 파일 시스템과 포맷 호환성

### NTFS (Windows 표준)
- **Windows**: 읽기·쓰기 완전 지원
- **macOS**: 읽기만 가능, **쓰기 불가능**
- **특징**: 파일 암호화, 권한 설정, 할당량 기능 지원
- **교차 플랫폼 사용**: 비권장. 맥 사용자가 파일을 수정할 수 없음.

### exFAT (교차 플랫폼 권장)
- **Windows**: 읽기·쓰기 지원
- **macOS**: 읽기·쓰기 지원
- **Linux**: 5.7 이후 지원
- **특징**: 파일 크기 제한 없음 (4GB 제한 없음), 모든 OS에서 쓰기 가능
- **권장**: Windows와 Mac을 함께 쓰는 경우 exFAT 선택.

### FAT32 (레거시·호환성)
- **모든 OS 호환**: Windows, macOS, Linux 모두 읽기·쓰기 가능
- **제한사항**: 
  - 단일 파일 크기 4GB 제한 (동영상, 설치 파일 불가)
  - 전체 용량 32GB 제한 (Windows 포맷 기준)
- **용도**: 매우 오래된 TV·카메라·게임기 호환이 필수일 때만.

**출처**: Lexar, Corsair 등 공식 가이드

## 4. macOS에서 NTFS 드라이브 쓰기 문제

### 문제 상황
macOS는 NTFS 포맷 드라이브에 기본적으로 **쓰기를 지원하지 않음**. 따라서 파일을 복사하거나 수정할 수 없고, 읽기만 가능.

### 해결책 1: 재포맷 (데이터 손실)
드라이브를 exFAT 또는 APFS로 재포맷하면 쓰기 가능해지지만, **모든 데이터가 삭제됨**.
**반드시 먼저 데이터를 백업할 것.**

### 해결책 2: 타사 NTFS 드라이버 설치 (유료)
- Paragon NTFS (유료)
- Tuxera NTFS (유료)
- Mounty (무료, 기능 제한)

포맷 없이 NTFS 쓰기가 가능하지만 소프트웨어 구입 비용 발생.

### 해결책 3: 권한 설정 변경
Finder에서 드라이브 우클릭 > "정보" > "Sharing & Permissions" 섹션 > "Ignore ownership on this volume" 체크. 효과가 제한적이지만 시도해볼 가치 있음.

**출처**: Apple Support, EaseUS, 타사 솔루션

## 5. 물리적 고장 신호 — 즉시 중단 경고

### 딸깍 소리 (반복되는 '딸깍, 딸깍')
**이것은 헤드 손상의 명확한 신호다.**
- 읽기 헤드가 플래터를 긁고 있다는 뜻
- 물리적 손상으로 **소프트웨어 복구 불가능**
- 추가 연결 시도는 손상을 악화시켜 전문 복구를 더 어렵게 함

**대처 방법:**
1. 즉시 드라이브 사용 중단
2. 어떤 복구 소프트웨어도 실행하지 말 것
3. "다시 한 번 해보기" 시도 금지
4. 데이터가 중요하다면 **전문 데이터복구 업체에 의뢰만 가능**

### 기타 이상 음성
"삐삐", "띠띠" 같은 비프음이나 긁히는 음이 난다면 역시 물리적 고장. 같은 방식으로 대처.

**출처**: 7 Data Recovery, 데이터복구 전문가 상담

---

## 6. 포맷 경고

### "포맷하시겠습니까?" 창이 나타났을 때
**절대로 포맷 버튼을 누르지 말 것.**
- 포맷을 진행하면 **모든 데이터가 완전히 삭제됨**
- 데이터 보존이 필요하면 반드시 복구 소프트웨어 사용 또는 전문가 상담

### RAW 파일시스템
디스크 관리에서 드라이브가 "RAW" 상태로 표시되면 파일시스템이 손상된 상태. 
- 포맷 전에 복구 소프트웨어로 스캔할 것
- 원본 상태의 디스크 이미지 백업 후 진행

## 7. 인식은 되는데 폴더를 열 수 없음

### 원인
1. **드라이브 문자 미할당**: 장치관리자에는 보이지만 내 PC에 드라이브 문자 없음
   → 디스크 관리에서 드라이브 문자 할당 (위의 Windows 단계 4 참조)

2. **파일시스템 손상**: 디스크 관리에서는 보이지만 폴더 열기 실패
   → 복구 소프트웨어(Recuva, EaseUS Data Recovery 등) 사용 고려
   → 포맷 전에 복구 시도

3. **권한 문제** (macOS): 파일 접근 권한 부족
   → 드라이브 정보 > Sharing & Permissions 확인

## 8. USB 3.0 호환성

### 버전 간 호환성
- USB 3.1 포트에 USB 3.0 기기 연결 가능 (역호환성)
- USB 2.0 기기도 3.0 포트에서 작동
- 단, 전송 속도는 **낮은 쪽 기준**

### 포트 색상 구분 (참고용)
- USB 2.0: 검정색, 흰색
- USB 3.0: 파란색
- USB 3.1 Type-C: 적색 또는 검정색
**주의**: 색상은 제조사 기준이지 표준이 아님. 정확한 정보는 컴퓨터 매뉴얼 또는 BIOS 확인.

### 케이블·허브 선택
고속 전송이 필요하면 **USB-IF 공식 인증 마크**(SuperSpeed+ 등)가 있는 제품 선택.
저가 미인증 제품은:
- 명시된 속도 미지원
- 발열 발생
- 전송 오류 → 데이터 손실 위험

**출처**: Kingston Technology, 커뮤니티 보고

## 9. 안전한 제거 중요성

### 절차
**Windows**: 시스템 트레이 > "안전하게 하드웨어 제거" > 드라이브 선택 > "중지"
또는 파일 탐색기에서 드라이브 우클릭 > "꺼내기"

**macOS**: Finder 사이드바에서 드라이브 옆 "꺼내기" 버튼 클릭

### 왜 중요한가?
- Windows/macOS는 쓰기 캐시를 사용하여 성능 최적화
- 안전한 제거 없이 급분리하면 **쓰기 중인 데이터가 손실**될 수 있음
- 특히 파일 복사 중일 때 위험

### 급분리 후 재연결 실패 시
파일시스템 메타데이터 손상 가능성. 복구 소프트웨어 사용 또는 전문가 상담.

**출처**: Microsoft Learn

## 10. 고급 해결책 체크리스트

1. **재연결**: 케이블 분리 > 1분 대기 > 재연결
2. **다른 포트**: 컴퓨터 다른 포트 시도
3. **다른 PC**: 다른 컴퓨터에서 테스트
4. **드라이버 재설치**: 장치관리자 > USB 컨트롤러 > 제거 > 재시작
5. **전원 관리**: Windows 전원 옵션 > USB 선택적 일시 중단 > 사용 안 함
6. **시스템 업데이트**: Windows 업데이트 확인
7. **칩셋 드라이버**: 마더보드 제조사 웹사이트에서 최신 칩셋 드라이버 설치

위의 모든 절차 후에도 인식 안 되고 물리적 손상 신호(딸깍 소리) 없으면, **전문가 점검 권장**.

## 조사 범위 및 한계

- **커버**: 인식 불가(하드웨어·드라이버 문제), 파일시스템 호환성, 물리적 고장 신호 판별
- **미커버**: 개별 브랜드 제품별 펌웨어 업데이트, 데이터 복구 소프트웨어 사용법 상세 (기술 깊이)
- **주의**: 물리적 고장 신호(딸깍 소리) 판단 후 복구는 전문 업체만 가능. 개인 시도 금지.
