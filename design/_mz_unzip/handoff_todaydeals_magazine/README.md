# Handoff: 오늘의딜 매거진 — 리디자인 (확정안)

## Overview
오늘의딜 매거진(`todaydeals.co.kr/magazine`) **메인 페이지의 확정 리디자인**. 기존 라이브는 오렌지가 마스트헤드/하단을 가득 채우는 면(面)으로 쓰여 답답했음 → **오렌지를 진짜 포인트로만** 쓰고, 따뜻한 페이퍼 배경 + 에디토리얼 세리프 헤드라인 + 헤어라인 룰 시스템으로 격상. 콘텐츠·브랜드·5코너 구조는 그대로.

**구현 대상 파일:** `오늘의딜 매거진 - 리디자인.dc.html` (이 README 옆). 브라우저로 바로 열려 시안 확인 가능(미리보기 런타임 `support.js` 동봉, 구현 시 무시). `.dc.html`은 **디자인 레퍼런스** — 그대로 배포하지 말고 기존 Next.js 앱의 `/magazine`에 **이 CSS/레이아웃으로 재구현**.

## Fidelity
**High-fidelity.** 색·폰트·간격·룰 두께·정렬을 정확히 재현.

---

## ⚠️ 이번 리디자인의 핵심 원칙 (반드시 반영)

1. **오렌지는 면이 아니라 점.** `#FF5A3C`는 로고·핵심 어절 강조·코너 점·"가이드 읽기" 밑줄·전체가이드 버튼·하단 강조어에만. **배경 전체를 오렌지로 칠하지 말 것.** 배경은 웜 페이퍼 `#F4EFE7`.
2. **콘텐츠 폭을 시원하게.** 기존 구현이 컬럼이 너무 좁아 "옹졸"했음. 본문 컨테이너 **`max-width:1180px`**, 좌우 패딩 40px, 요소 간 넉넉한 여백. 좁은 박스에 가두지 말고 넓게 펼칠 것. ← **중요**
3. **룰(선)은 얇고 부드럽게.** 진한 2px 블랙 금지. 모든 구분선 **`1px solid rgba(22,20,15,0.16)`**(보조선은 `0.08`). 잉크 블랙 면(푸터·중립밴드)만 진하게.
4. **헤드라인은 세리프, 그 외는 Pretendard.** 위계로 메시지를 전달(아래 타이포 참고).

## 페이지 구조 (위→아래)

1. **유틸 바**(잉크 배경): 좌 "● 광고 0 · 제휴 0 · 구매 판단의 기준만", 우 [오늘의딜 홈 · 매거진]. ※ 핫딜게시판 링크 없음(의도적 제외).
2. **마스트헤드 헤더**(페이퍼, 하단 1px 룰): 로고(시계+오늘의딜.+MAGAZINE) / 코너 5 내비(밑줄 스윕 호버).
3. **슬로건 섹션**: eyebrow(MONO) → 위계 H1(위 참고). 풀폭, 좁은 컬럼 금지.
4. **코너 인덱스**(상단 1px 룰): 5칸 그리드 `01–05` + 코너명 + 영문 + 한 줄 설명 + 코너 점. 호버 시 잉크 반전.
5. **피처드**: 2단 그리드 `1.15fr 1fr`, **`align-items:start`(상단 정렬 — 좌 eyebrow가 우 이미지 상단과 같은 높이에서 시작)**. 좌: 코너칩+분야+FEATURED → 세리프 제목 → 리드 → "가이드 읽기"(오렌지 밑줄)+메타. 우: 4:3 이미지 슬롯.
6. **최신 가이드**(상단 1px 룰): 에디토리얼 리스트 행 `[번호 42px][코너 110px][세리프 제목 1fr][화살표]`, `align-items:center`. **코너셀 정렬 주의:** 점을 텍스트에서 분리해 `flex; align-items:center; gap:9px` → [점] + [코너명/분야명 2줄 컬럼]. 점이 두 줄 가운데 정렬되고 두 줄 좌측 끝이 일치(점을 코너명 앞에 붙이지 말 것).
7. **중립 선언 밴드**(잉크 배경): 원형 체크(오렌지) + 세리프 "정답 대신 기준을, 권유 대신 근거를 드립니다" / 서브 "무언가를 고르고 결정한다는 것은 생각보다 외롭습니다 · 당신은 혼자가 아닙니다(오렌지)" + **얇은** 오렌지 버튼 "전체 가이드 →"(padding `11px 24px`, radius 8px).
8. **푸터**(딥 잉크): 좌 [오늘의딜 홈 · 매거진 · 이용약관 · 개인정보처리방침], 우 "© 2026 TODAYDEALS. All rights reserved."

## 상세(아티클) 페이지 — `오늘의딜 매거진 - 상세 기준안.dc.html`

기존 클로드코드 구현은 본문을 **좁은 읽기 컬럼**에 가두고, 표·콜아웃만 더 넓어 좌우 선이 어긋났음(폭 불일치 = 결함). 이 기준안이 올바른 정렬·폭의 **정답지**다.

**핵심 원칙 — 본문도 풀폭으로:**
- 디자인은 시원한데 콘텐츠만 좁게 끼우지 말 것. 아티클도 메인과 동일하게 **컨테이너 `max-width:1120px`, 좌우 패딩 44px**를 쓴다.
- 본문은 **2단 그리드**: `grid-template-columns:1fr 320px; gap:64px; align-items:start`.
  - 좌 = **메인 본문**(단락·소제목·비교표·체크리스트·비용 막대). 모든 요소가 메인 컬럼의 좌·우 선을 공유 — 표/박스가 본문보다 튀어나오지 않게.
  - 우 = **사이드 레일**(`position:sticky; top:24px`): AD-FREE 정직성 배지 · 3줄 요약 · "짚고 가요" 박스.
- 타이틀 블록과 "정직한 마무리"(잉크 카드)는 **풀폭**(그리드 밖).
- 반응형: `@media (max-width:900px)` → 그리드 1열, 레일 `position:static`(본문 아래로).

**구성 순서:** 유틸바 → 헤더 → [타이틀 블록(브레드크럼·코너칩·세리프 H1 46px·세리프 데크·메타, 하단 1px 룰)] → [2단 그리드: 메인(리드 → "광고 문구 vs 실제 의미" 비교표 → 면적/소음/유지비 섹션 → 3년 비용 막대 → "사기 전 5줄 체크리스트") + 레일(배지·3줄요약·짚고가요)] → 풀폭 "정직한 마무리"(잉크) → 푸터.

**비교표 스펙:** 2열 그리드 `0.85fr 1.15fr`, 잉크 헤더(흰 글씨, 열 사이 `rgba(255,255,255,0.14)` 세로 룰), 행 줄무늬 `#faf8f5`, 좌/우열 사이 `rgba(22,20,15,0.08)` 세로 룰 + 행 구분 1px. 우열 `line-height:1.75`로 가독성 확보.

**유지비/계산형:** "공개 스펙 기준 추정" 면책 필수(§9 안전선). 막대는 [본체값 + 3년 필터값] 스택.

> 요약: **부품(컴포넌트)은 좋았고 조립(그리드)이 틀렸던** 문제를 이 기준안이 바로잡는다. 본문·표·박스·소제목의 좌우 시작선을 한 컬럼으로 통일하고, 남는 폭은 사이드 레일로 의미 있게 채울 것.
