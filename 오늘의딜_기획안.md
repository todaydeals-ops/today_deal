# 오늘의딜 (Oneuldeal) — 기획안 & 개발 사양서

> **매일매일 새로운 타임딜을 한눈에**
> 국내외 주요 쇼핑몰의 타임딜을 한 곳에 모으고, 제휴 수익을 창출하는 핫딜 큐레이션 플랫폼
>
> 문서 버전: v1.0 · 작성일: 2026.06
> 용도: 사업 기획 정리 + Claude Code 개발 핸드오프

---

## 1. 프로젝트 개요

### 1.1 한 줄 정의
지마켓·11번가·알리익스프레스의 **타임딜을 자동 수집**해 통일된 UI로 보여주고, **제휴 커미션**으로 수익을 내는 사이트.

### 1.2 운영 주체
- 슬로우베리와 **분리 운영**되는 독립 브랜드 (별도 BI)
- 쇼츠·네이버클립에서 활동하는 **쿠팡 파트너스 상위 파트너(가족)와 제휴**해 고전환 트래픽 확보

### 1.3 수익 구조 (우선순위)
1. **1순위 — 제휴 커미션**: 파트너 고전환 트래픽 기반 쿠팡·각 쇼핑몰 제휴 수익
2. **2순위 — 회원 DB**: 무료나눔으로 확보한 회원·마케팅 동의 → 향후 네이버쇼핑 등 배너 광고 영업 자산

### 1.4 핵심 차별점
- 타 핫딜 사이트의 "떨이판" 인상을 지운 **소프트 뉴트럴 디자인** (큐레이션된 살림 정보 톤)
- 각 딜마다 **최저가 비교** 기능 (쿠팡 등과 가격 대조)
- **3사 타임딜 통합** — 흩어진 타임딜을 한 화면에서 비교

## 2. 사이트 구조 (Information Architecture)

```
오늘의딜
├── 타임딜          → 지마켓·11번가·알리 3사 실시간 타임딜 (메인)
├── 추천딜          → 가족 파트너가 직접 고른 쿠팡 큐레이션 핫딜
└── 나눔딜          → 무료나눔 이벤트 (회원 전용, DB 확보 엔진)
```

### 2.1 메뉴 네이밍 원칙
세 메뉴 모두 "딜"로 끝나 브랜드 운율 통일. "쿠팡" 등 특정 플랫폼명은 메뉴에 노출하지 않음 (확장성·의존도 인상 완화).

## 3. 카테고리별 상세 기획

### 3.1 타임딜 (메인)

#### 데이터 소스
| 플랫폼 | 딜 명칭 | URL | 수집 방식 |
|---|---|---|---|
| 지마켓 | 슈퍼딜(타임딜) | `gmarket.co.kr/n/superdeal` | 웹 크롤링 |
| 11번가 | 쇼킹타임 | `deal.11st.co.kr/browsing/DealAction.tmall?method=getTimeDeal` | 웹 크롤링 (EUC-KR) |
| 알리익스프레스 | 타임딜(Flash Deal) | `aliexpress.com/ssr/300001014/Flashdeal` | Affiliate API 우선, 크롤링 보조 |

#### 갱신 주기 (플랫폼별 맞춤)
| 플랫폼 | 딜 오픈 주기 | 수집 스케줄 |
|---|---|---|
| 11번가 | 매시간 정각 | 매시간 정각 + 5분 |
| 지마켓 | 매일 (오전) | 매일 오전 10:30 |
| 알리 | 매일 오전 10시 | 매일 오전 10:40 |
| **전 플랫폼 공통** | — | **30분마다 품절 여부만 재확인** |

#### 수집 규칙 (핵심)
> **살아있는 카운트다운 타이머가 있는 카드 = 진짜 타임딜.** 예정/종료 상태는 모두 제외.

| 플랫폼 | 수집 대상 | 제외 |
|---|---|---|
| 지마켓 | 타임딜 카운트다운 타이머 있는 카드 | 오늘의 쇼츠딜 (타이머 없음) |
| 11번가 | "쇼킹 타임딜" + 카운트다운 | "판매예정"(가격 미공개), 일반 상품 |
| 알리 | "초특가 타임딜" + 종료 타이머 | "내일의 특가"(예정) |

#### 수집 데이터 항목
```
필수:
- 상품명 (product_name)
- 상품 이미지 URL (image_url)
- 할인율 % (discount_rate)
- 판매가 (sale_price)
- 마감 시각 / 남은시간 (deal_end_at)
- 상품 상세 URL (product_url) → 제휴 링크 변환용
- 플랫폼 (platform: gmarket/11st/ali)

선택(있으면 저장, 화면 비노출):
- 정가 (original_price)
- 무료배송 여부 (free_shipping)

비수집 (의도적 제외):
- 판매량 / 구매수 / 판매진행률
  → 딜 오픈 시점 스냅샷이라 시간 지나면 부정확. 표시하지 않음.
- 품절 여부만 30분 주기로 갱신 (is_soldout: boolean)
```

#### 레이아웃
- **세로 3열** (지마켓 / 11번가 / 알리), 각 열에 플랫폼이 노출한 순서대로 카드 세로 스택
- 재정렬 없음 (플랫폼 자체 알고리즘 순서 신뢰), 수량 맞추기 없음 (한두 개 차이는 무시)

### 3.2 추천딜 (쿠팡 큐레이션)
- 가족 파트너의 고전환 트래픽이 연동되는 핵심 수익 페이지
- 관리자가 직접 등록(수동 큐레이션) — 쿠팡 파트너스 API로 링크·정보 보조
- 전환율이 핵심이므로 "갖고 싶게 만드는" 상품 선별 중심

### 3.3 나눔딜 (무료나눔)
- **목적**: 회원가입 + 마케팅 동의 확보 (2순위 수익 자산)
- **운영**: 예산 책정해 경품 직접 지급. 초기 주 10명 → 확대 시 주 30명
- **응모 조건**: 회원가입 + 마케팅 수신 동의 (쿠팡 클릭과 분리)
- **경품 페이지**: 상품 소개·스펙 등 콘텐츠 실체 확보 (정상 영업 페이지 요건)

#### 회원가입 설계
- 간편가입 (카카오/네이버)
- **마케팅 수신 동의는 필수 동의와 분리된 별도 체크박스** (정보통신망법 준수)
- 확보 목적: 카카오 플친 추가 유도, 향후 배너 영업 시 "활성 회원 N명" 근거

## 4. 핵심 기능 — 최저가 비교

각 딜 카드의 **[최저가 비교]** 버튼.

### 4.1 동작
- 클릭 시 해당 상품을 **쿠팡(+다나와 등)에서 키워드 검색**한 결과로 연결
- 각 결과 링크에 해당 플랫폼 **제휴코드** 삽입
- 상품명이 플랫폼마다 달라 정확 매칭은 어려움 → **키워드 검색 결과 연결**이 현실적

### 4.2 품절 상태일 때
- 버튼이 **"쿠팡에서 비슷한 상품 찾기"**로 변경
- 놓친 아쉬움을 쿠팡 트래픽으로 전환하는 동선

### 4.3 정책 준수
- 쿠팡을 타 쇼핑몰과 함께 가격비교로 묶는 것은 **정책 위반 아님** (시장 표준)
- 단, 수수료 고지 문구 필수 노출 (푸터 + 비교 결과 화면)

## 5. 수익화 & 제휴 처리

### 5.1 제휴 링크 처리 (정공법)
```
[수집] 상품 데이터만 우리 DB에 저장
   ↓
[렌더링] 우리 디자인으로 카드 표시
   ↓
[이동] 버튼 클릭 → 우리 제휴 링크로 상품 페이지 이동
```
> iframe 임베드·클릭재킹 등은 사용하지 않음 (약관 위반·계정 정지 리스크).

### 5.2 제휴코드 연동
| 플랫폼 | 제휴 방식 |
|---|---|
| 쿠팡 | 쿠팡 파트너스 API |
| 알리 | Affiliate API |
| 지마켓·11번가 | 텐핑 등 제휴 네트워크 경유 |

### 5.3 필수 고지 (법적 의무)
```
이 사이트는 쿠팡 파트너스 활동의 일환으로,
이에 따른 일정액의 수수료를 제공받습니다.
```
- 위치: 푸터 상시 + 최저가 비교 결과 화면
- 누락 시 약관 위반으로 계정 정지 사유

## 8. 개발 사양 (Claude Code 핸드오프)

### 8.1 권장 스택
> 슬로우베리 기존 스택과 일관성 고려

- **프론트엔드**: Next.js + React
- **백엔드/DB**: Supabase (PostgreSQL)
- **크롤러**: Python (BeautifulSoup/Playwright) 또는 Node
- **스케줄러**: Vercel Cron
- **배포**: Vercel
- **인증**: Supabase Auth (카카오/네이버 소셜)
- **이메일**: Resend (마케팅 발송)

### 8.2 DB 스키마 (초안)

```sql
-- 타임딜 상품
deals (
  id              uuid primary key,
  platform        text,        -- 'gmarket' | '11st' | 'ali'
  product_name    text,
  image_url       text,
  product_url     text,        -- 원본 상품 URL
  affiliate_url   text,        -- 제휴코드 삽입된 링크
  discount_rate   int,
  sale_price      int,
  original_price  int,
  free_shipping   boolean,
  deal_end_at     timestamptz, -- 마감 시각
  is_soldout      boolean default false,
  collected_at    timestamptz, -- 수집 시각
  display_order   int          -- 플랫폼 노출 순서
)

-- 추천딜 (수동 큐레이션)
curated_deals (
  id, product_name, image_url, affiliate_url,
  discount_rate, sale_price, admin_note, created_at, is_active
)

-- 회원
users (
  id uuid primary key,
  provider text,               -- 'kakao' | 'naver'
  email text,
  marketing_consent boolean,   -- 마케팅 수신 동의 (분리)
  consent_at timestamptz,
  created_at timestamptz
)

-- 무료나눔 이벤트 / 응모
giveaways (id, title, prize_name, prize_image, start_at, end_at, winner_count)
giveaway_entries (id, giveaway_id, user_id, entered_at)
```

### 8.3 크롤러 모듈 구조
```
crawler/
├── gmarket.py     # 슈퍼딜 타임딜 수집 (타이머 있는 카드만)
├── elevenst.py    # 쇼킹타임 수집 (EUC-KR 인코딩 주의, 판매예정 제외)
├── ali.py         # 타임딜 수집 (Affiliate API 우선)
├── soldout.py     # 30분 주기 품절 체크 (공통)
└── scheduler.py   # 플랫폼별 갱신 주기 관리
```

### 8.4 크롤링 주의사항
- **지마켓**: robots/약관상 스크래핑 제한 명시. 차단 대응(User-Agent, 간격 조절, Playwright) 필요. 차단 시 텐핑 어드민 데이터 활용 대안 검토.
- **11번가**: EUC-KR 인코딩 → UTF-8 변환 필수
- **알리**: Affiliate API 발급 2~3일 소요. 미리 신청.
- 전 플랫폼 과도한 요청 시 IP 차단 → 적절한 간격·헤더 설정

### 8.5 화면별 컴포넌트
```
components/
├── Header.tsx          # 로고 + 네비(타임딜/추천딜/나눔딜) + 로그인
├── Banner.tsx          # 배너 영역 (이미지 자율 크기)
├── DealCard.tsx        # 타임딜 카드 (6슬롯, 품절 상태 포함)
├── DealColumn.tsx      # 플랫폼별 세로 열
├── CompareButton.tsx   # 최저가 비교 버튼
├── Countdown.tsx       # 카운트다운 타이머
└── Footer.tsx          # 제휴 고지 + 면책
```

## 9. 운영 원칙

### 9.1 데이터 신뢰
- 부정확할 수 있는 데이터(판매량 등)는 **표시하지 않음**. 틀린 정보보다 없는 게 낫다.
- 품절은 30분 주기로 갱신해 헛클릭 방지.

### 9.2 트래픽 전략
- 핵심 무기는 SEO·입소문이 아니라 **가족 파트너의 고전환 트래픽 연동** (전환율 30%대)
- 이름·디자인은 "기억하기 쉽고 신뢰감 있는" 수준이면 충분, 승부는 전환·재방문

### 9.3 컴플라이언스
- 마케팅 수신 동의 별도 분리 (정보통신망법)
- 제휴 수수료 고지 상시 노출 (쿠팡 파트너스 약관)
- iframe·클릭재킹 등 약관 위반 기법 미사용
