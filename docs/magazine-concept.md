# 오늘의딜 매거진 — 컨셉 기획서

> 쇼핑 의사결정을 돕는 **중립 쇼핑 전문 웹진**. (오늘의딜 딜 사이트와 별개 섹션)
> 이 문서 하나로 컨셉·문체·구조·디자인 요구사항을 모두 파악할 수 있습니다.

---

## 0. 한 줄 정의
**"안 팔고, 안 까고, 안 빠는"** — 광고·제휴 없이 **구매 판단의 기준만** 주는 쇼핑 가이드 매거진.
딜 사이트가 "지금 싸요, 사세요"라면, 매거진은 **"이건 이렇게 고르세요, 광고는 이게 거짓이에요"**.

## 1. 핵심 철학 (절대원칙)
- **중립**: 특정 제품을 추천하지도, 비방하지도 않는다. 판단은 소비자의 몫.
- **무광고·무제휴**: 본문에 구매/제휴 링크 **0**. 돈을 안 받으니 믿는다 = 신뢰의 해자.
- **도움만 준다**: 기준·트레이드오프·실제 비용을 제시하고 물러선다.
- 이 철학이 흔들리면 매거진의 존재 이유가 사라짐.

## 2. 캐치프라이즈 (모든 글 상단 고정 배지)
> **이 글은 끝까지 읽어도 특정 제품을 추천하지 않습니다.**
> **소비자의 바른 선택, 그 방향만 제시합니다.**

- 슬로건(마스트헤드용): **"팔지 않습니다, 방향만 드립니다."**
- 매 글 최상단에 체크 아이콘 + 이 문구를 **작은 배지/박스**로 자동 삽입.

## 3. 콘텐츠 구조 — 5 코너 × 분야 태그

**코너(형식) 5종** — 매거진의 브랜드 섹션:
| 코너 | 약속 | AI 무기 |
|---|---|---|
| 🔍 **팩트체크** | 광고 너머의 진실·숨은 단점 | 후기 패턴 분석(정성적, 통계 날조 금지) |
| 🧭 **스마트가이드** | 내 상황엔 뭘 골라야? | Decision Tree(YES/NO 선택형) |
| 📊 **끝장비교** | A vs B, 3년 총비용까지 | 유지비 시뮬레이션 매트릭스(공개 스펙 계산) |
| 🛠 **롱런팁** | 본전 뽑고 오래 쓰는 법 | 활용·관리 실전 |
| 🧪 **트렌드랩** | 신기술, 살 가치 있나? | 빠른 검증 |

**분야 태그(크로스)**: 가전 · 리빙/주방 · 디지털/IT · 식품/건강 · 뷰티 · 유아/반려

→ 주제 = **분야 × 코너** 매트릭스의 빈칸. 예) "리빙/주방 × 끝장비교 = 음식물처리기 방식 비교".

## 4. AI 차별화 (영혼 없는 AI글 탈출)
누구나 AI로 쓰는 시대 → **나열식은 묻힘**. AI의 고유 강점으로만 차별화:
- **유지비 시뮬레이션 매트릭스**: 소비전력·필터값·교체주기 → 3년 총비용 계산(정량·정직·베끼기 어려움) ← **본진**
- **Decision Tree**: "하루 3끼 다 먹나? Y→미생물 / 주말몰아서? Y→건조분쇄" YES/NO 인포그래픽
- **비교표/체크리스트**: 한눈에 보는 종결형 표 (공유 자산화)

## 5. 화자(문체) — 7대 규칙
**페르소나**: 소비자 편에 선 깐깐한 쇼핑 분석가. 안 팔고, 근거와 트레이드오프만 주고 물러선다.
**롤모델 문체**: 노써치(nosearch.com)식 중립 가이드 — 공감 도입 → 객관 기준 → 수치·표 → 판단은 독자.

1. **공감 도입** — 독자의 진짜 고민으로 연다.
2. **중립 절대원칙** — "사세요/최고/사지마세요" 금지. **조건부 결론**: "A 상황이면 X, B면 Y".
3. **근거·수치 우선** — 광고문구 X → 공개 스펙·계산.
4. **AI 차별화** — 유지비 표·Decision Tree·비교 매트릭스 적극.
5. **정직한 마무리** — 한계·현실 짚고 **"선택은 당신의 몫"**. 밀어붙이지 않음.
6. **문장** — 통일 존댓말(~합니다), 중간 길이, 읽기 쉬운 리듬. 전문성+친근.
7. **무광고·무제휴** — 본문에 구매/제휴 링크 0. "어디서 싸게"는 안 다룸(그건 딜 사이트).

## 6. 제작 워크플로 (편집국)
| 역할 | 단계 |
|---|---|
| **총괄팀장** | 주제 선정 · 팩트체크 게이트 · 최종 컨펌 |
| **주제발굴** | 분야×코너 빈칸 + 검색 페인포인트 발굴 |
| **자료조사 ×2** | 서로 다른 앵글로 조사(실용 / 스펙·데이터) + 교차검증 |
| **글쓰기** | 위 화자 보이스로 종합·구조화 |
| **이미지** | CI 템플릿 썸네일 + 비교표·Decision Tree 시각화 |
| (영상 — V3, 나중) | 짧은 영상 → 유튜브 → 임베드 |

- **AI 80%는 취합·포맷팅, 사람(총괄/대표) 20%는 기획·팩트체크.**
- **발행 전 사람 검증 필수** (특히 수치·비교).

## 7. 운영 원칙
- **하루 1편 원칙.** 발행 트리거 = **세션 기반**(작업 창을 열면 한 편 가동, 쉬면 쉼 → 유휴비용 0, 품질 통제).
- **에버그린 80% / 트렌드 20%** — 검색 트래픽 복리 + 화제성.
- 양치기 금지(구글 스케일드 콘텐츠 페널티). **주 몇 편이라도 제대로**.

## 8. 수익 구조 (정직하게)
- 매거진 본문 = **제휴 0** (신뢰 자산).
- 수익은 → ① 애드센스(중립 디스플레이) ② 브랜드 후광 → 오늘의딜 딜 사이트 자연 유입 ③ 충성 트래픽 자체.
- 매거진 = **신뢰·트래픽 엔진**, 돈은 딜 사이트/애드센스가. 분리가 둘 다 키움.

## 9. 안전·금지선 (지키지 않으면 사업이 망함)
- ❌ **통계 날조 금지**: "73%가 지적" 같은 숫자를 데이터 없이 만들면 명예훼손·표시광고법·구글 페널티. → 정성적 표현("후기에서 반복되는 불만은…")으로.
- ❌ **리뷰 대량 스크래핑 금지**: 쿠팡·네이버 후기 긁기 = 차단·약관 위반.
- ✅ **계산형 콘텐츠는 안전**: 공개 스펙으로 유지비 계산 = 정직·강력.
- ✅ **사실은 출처 기반만**: 제조사 공식 스펙·매뉴얼.

## 10. URL / 기술
- URL: **`todaydeals.co.kr/magazine`** 또는 **`/guide`** (서브디렉터리 — SEO 유리, 기존 Next.js 앱에 섹션 추가). ※ 한글 서브도메인 비추(punycode).
- 저장: Supabase 테이블 + 글 페이지(메타·Article JSON-LD·sitemap). 분야/코너 분류.
- 엔진: 클로드 기반 생성(오늘의딜 ANTHROPIC_API_KEY 재사용). SeoulEdits 엔진 패턴 차용.

## 11. 디자인 요구사항 (← 클로드디자인용)
- **CI 로고 기반** 비주얼 아이덴티티(오늘의딜 오렌지 #ff5a3c 계열).
- **정직성 배지**: 모든 글 상단 고정(체크 아이콘 + §2 캐치프라이즈). 눈에 띄되 담백하게.
- **코너별 시각 아이덴티티**: 5개 코너(팩트체크·스마트가이드·끝장비교·롱런팁·트렌드랩) 각각 아이콘·컬러 칩.
- **핵심 컴포넌트 템플릿**:
  - 비교표(끝장비교) — 종결형, 한눈에.
  - 유지비 매트릭스 — 3년 총비용 시각화.
  - Decision Tree(스마트가이드) — YES/NO 분기 인포그래픽.
  - 체크리스트 카드.
- **썸네일 템플릿**: CI 기반 + 제목 자동 얹기(코너 라벨 노출).
- 톤: 신뢰감·전문성·담백함. 광고 같지 않게. 가독성 최우선(긴 글이라 표·소제목·여백).
