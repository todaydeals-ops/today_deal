# 쓰레드(Threads) API 자동발행 셋업 가이드 — 직원 인계용

> 목적: 오늘의딜 핫딜/매거진 콘텐츠를 **쓰레드에 자동 발행**하기 위한 계정·앱·토큰 준비.
> 이 문서대로 끝까지 진행한 뒤, **마지막 "개발팀에 전달할 값" 4개**를 담당자에게 넘겨주세요.
> 검증 기준일: 2026-06. (메타 정책은 바뀔 수 있으니 화면 문구가 다르면 캡처해서 문의)

---

## 0. 미리 알아둘 것 (요약)
- 쓰레드 API는 **무료**입니다.
- **인스타그램 비즈니스 계정·페이스북 페이지 없어도** 됩니다(쓰레드 자체 로그인으로 진행).
- 단, **브랜드 인스타도 곧 운영 예정**이라 → **인스타 계정을 먼저 만들고 그 계정으로 쓰레드를 만드는 것**을 권장(한 핸들로 두 채널 동시 확보).
- 게시 한도: **하루 250건**(우리 용도엔 차고 넘침).
- 토큰은 **60일짜리 장기 토큰** → 만료 전 갱신 필요(개발팀이 자동 갱신 처리).

---

## 1. 계정 만들기

### 1-1. 브랜드 계정명(핸들) — 먼저 정하기
**모든 채널(쓰레드·인스타·네이버블로그)에서 같은 핸들로 통일**하세요. 브랜드 각인에 중요합니다.
- 표시 이름: **오늘의딜**
- 핸들 우선순위(가능한 것으로): `@todaydeals.kr` → `@todaydeals_official` → `@oneuldeal`
- 프로필 한 줄 소개(예): `광고 없는 중립 쇼핑 가이드 · AI가 검증하는 진짜 싼 딜`
- 프로필 링크: `https://www.todaydeals.co.kr`
- 프로필 이미지: 오늘의딜 로고(주황 시계 아이콘)

### 1-2. 인스타그램 계정 생성 (권장 선행)
1. https://www.instagram.com → 가입 → **업무용/비즈니스 계정**으로 전환
2. 위에서 정한 핸들·이름·소개·링크·로고로 세팅

### 1-3. 쓰레드 계정 생성
1. https://www.threads.net → **인스타 계정으로 로그인**(1-2에서 만든 계정)
2. 프로필 동일하게 정리

> 인스타 없이 가려면 쓰레드를 이메일로 단독 생성도 가능하나, **인스타 연동 방식 권장**(나중에 인스타 채널까지 한 번에).

## 2. 메타 개발자 계정 & 앱 만들기

### 2-1. 메타 개발자 등록
1. https://developers.facebook.com 접속 → 위 계정(또는 회사 대표 계정)으로 로그인
2. 우상단 **시작하기(Get Started)** → 개발자 등록(휴대폰 인증 등 안내대로)

### 2-2. 앱 생성
1. 상단 **내 앱(My Apps)** → **앱 만들기(Create App)**
2. 앱 이름: `오늘의딜 Threads` (내부용, 사용자에게 안 보임)
3. 유스케이스(Use case) 선택 화면에서 → **"Threads API" / "Access the Threads API"** 선택
   - 목록에 없으면, 앱 생성 후 대시보드에서 **제품 추가(Add Product) → Threads** 선택

### 2-3. 권한(스코프) 설정
앱 대시보드 → **Use cases → Customize(맞춤설정)** 에서 아래 권한 추가:
- `threads_basic` (필수 — 프로필 기본 조회)
- `threads_content_publish` (필수 — **게시**)

> (선택) 답글·인사이트까지 자동화하려면 `threads_manage_replies`, `threads_manage_insights`도 추가. 지금은 위 2개면 충분.

---

## 3. OAuth 리디렉션 URI 설정
앱 설정에서 콜백 주소를 등록해야 토큰을 받을 수 있습니다.
1. 앱 대시보드 → **Threads API 설정**(또는 Settings) 화면
2. **Redirect Callback URLs(유효한 OAuth 리디렉션 URI)** 에 아래 둘 다 입력:
   - 개발용: `https://localhost/auth/threads/callback`
   - 운영용: `https://www.todaydeals.co.kr/auth/threads/callback`
   > 정확한 콜백 경로는 개발팀이 최종 확정합니다. **일단 위 2개 등록**해두면 됩니다.

---

## 4. 액세스 토큰 발급
1. 앱 대시보드 → **Threads → Get Started / Generate Token(토큰 생성)**
2. 1-3에서 만든 **쓰레드 계정으로 권한 승인(Authorize)**
3. 처음엔 **단기 토큰(1시간)** 이 나옵니다 → **장기 토큰(60일)으로 교환**하는 버튼/절차 진행
   - (이 교환·이후 자동 갱신은 개발팀이 코드로 처리하므로, **단기 토큰까지만 받아도 OK**. 가능하면 장기 토큰까지.)

---

## 5. ✅ 개발팀에 전달할 값 (이게 최종 목표)
아래 4가지를 **안전하게**(메신저 평문 X, 비밀 메모/암호화) 전달:

| 항목 | 어디서 |
|---|---|
| **App ID (Client ID)** | 앱 대시보드 → 설정 → 기본 |
| **App Secret (Client Secret)** | 앱 대시보드 → 설정 → 기본 (표시 클릭) |
| **Threads 액세스 토큰** | 4단계에서 발급(장기 토큰이면 더 좋음) |
| **Threads 사용자 ID** | 토큰 발급 화면 또는 `/me` 조회값(개발팀이 토큰으로 확인 가능) |

추가로 알려줄 것:
- 쓰레드 **계정 핸들**(@…)과 로그인 정보(별도 안전 경로)
- 등록한 **콜백 URL**

## 6. 주의사항
- **게시 한도 250건/일** — 우리는 하루 몇 건이라 무관.
- **장기 토큰 60일 만료** → 개발팀이 자동 갱신 코드 적용(끊기면 발행 멈춤).
- 메타 정책상 **자동 스팸성 대량 발행 금지** — 우리는 하루 소량 양질이라 문제없음.
- 화면 UI/문구가 이 문서와 다르면 **그 단계 캡처**해서 공유(메타가 자주 개편함).
