# 콘텐츠 발행 자동화 원칙 (맥스↔API 라우팅)

## 큰 원칙
> **창 열림 = 맥스(Claude Code 세션)로 ‘밀린 것까지 최대 발행’ / 창 닫힘 = API로 ‘죽은 느낌만 면할 최소 발행’.**

- 맥스플랜은 고정비(쓰든 안 쓰든 동일) → 창 열려 있을 때 최대 활용해 **API 종량 과금을 최소화**.
- 수집(크롤)과 발행(생성)을 **분리**: 딜은 계속 크롤해 **대기 큐**에 쌓고, 큐를 빼는 속도만 모드별로 다르게.
  → 창 닫힘 동안 놓치는 딜 없이 큐에 쌓였다가, 창 열면 한 번에 따라잡음.

## 모드 감지 — 하트비트
- `automation_status` 한 줄(`source='local'`, `updated_at`).
- 창 열림 동안 로컬(나)이 주기적으로 `updated_at` 갱신.
- 서버는 생성 전 하트비트 확인: **신선(<15분)하면 건너뜀**(맥스가 처리 중), **낡으면 API 생성**.

## 모드별 발행량 (기본값 — 조정 가능)

| 콘텐츠 | 🟢 창 열림 (맥스·나) | 🔴 창 닫힘 (API) |
|---|---|---|
| 게시판 핫딜 글(리라이트) | 큐 **전량 소진**, 5분마다 배치 | **시간당 2~3건**만 신규 |
| 딜 한줄평(blurb) | `summary` 빈 것 **전량 백필** | 노출 딜만 **회당 3~5건** |
| 매거진 | 정상(하루 1편 + 요청 시 +N) | **발행 안 함**(맥스 전용) |

## API 안전장치 (과금 폭주 방지)
- 일일 상한: 게시글 **40건**, 한줄평 **60건**. 도달 시 그날은 큐 보관.
- 새벽 **0~7시 API 발행 정지**(트래픽 적고, 창 열면 채워짐).
- "죽은 느낌만 안 나게" 기준 = 영업시간대 **시간당 새 글 2~3개**.

## 구현 메모
- `lib/automation/heartbeat.ts` — isLocalAlive / touchHeartbeat
- `lib/automation/policy.ts` — 위 숫자 상수
- board ingest는 즉시 생성하지 않고 **pending 적재**, 드레이너가 모드별 속도로 소진
- API 드레인 크론: 하트비트 게이트 + 시간당/일일 상한 + 새벽 정지
- 로컬 드레인 스크립트: 내가 큐 전량 소진 + 하트비트 갱신
